Five surfaces the brand overhaul didn't reach yet, plus two ambient components. None are spec-defined — they predate the rebrand and survived because they sit outside the conversation flow. The treatment below applies the same restraint discipline as the rest of the product: workspace canvas (or brand canvas for the one celebratory moment), Bricolage Grotesque, brass for action color and brand-punctuation, sharp corners, meta-style trust signals.

Order recommendation. IdReveal first (one-time but high-impact brand moment), then CallsScreen (in the daily Calls tab), then InAppBanner (high-frequency, low surface area). Settings and GifPickerSheet are minor cleanup, mostly font/radius swaps. Diagnostics last because it's internal-only — the existing styling works, this is just consistency.

What's not here. No tab bar redesign — the bottom Chats/Calls tabs already use brand tokens through the existing tab navigator config. No splash / loading screen — RN's native splash is the launch experience and isn't a JS surface. No error screen — Sentry / global ErrorBoundary work happens elsewhere.

Server-touching gaps remain unfulfilled. The block flow's privacy property still needs server-side enforcement; group lifecycle system messages still need server broadcasts; per-conversation push gating needs either a server hint or a native NSE. Those are unrelated to the rebrand pass — they live in the spec follow-up list.
